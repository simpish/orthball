## ピン割り当て (RP2040-Zero)

```
RP2040-Zero (表20ピン + 裏9パッド = 29GPIO)

【表面ピン (20本)】
├── キーマトリクス COL:  GP0, GP1, GP2, GP3, GP4, GP5  (6本)
├── キーマトリクス ROW:  GP6, GP7, GP8, GP9            (4本)
├── PMW3360 SPI0:       GP18(SCK), GP19(MOSI), GP16(MISO), GP17(CS)  (4本)
├── PMW3360 MOTION:     GP20                            (1本)
├── E-Ink SPI0共有:     GP21(CS), GP22(DC)              (2本)
├── ロータリーエンコーダ: GP26(A), GP27(B), GP28(SW)     (3本)
│
【裏面パッド (9本)】
├── E-Ink BUSY:         GP10                            (1本)
├── E-Ink RST:          GP11                            (1本)
├── タッチ I2C SDA:     GP12                            (1本)
├── タッチ I2C SCL:     GP13                            (1本)
├── タッチ INT:         GP14                            (1本)
├── タッチ RST:         GP15                            (1本)
└── 未使用:             GP23, GP24, GP25                (3本: LED/将来拡張)

使用: 26/29  余り: 3
```

> ※ ピン番号は仮。PCB設計時にSPIバス/I2Cバスの物理ピン制約に合わせて調整する
> ※ SPI0をPMW3360とE-Inkで共有（CSで切り替え）

## 技術メモ

### QMK E-Ink対応
- Quantum PainterにE-Inkドライバは未実装（2026-02時点）
- SSD1681のカスタムSPIドライバ作成が必要
- タッチコントローラ (GT911系) のI2Cドライバも自作

### PMW3360接続
- RP2040は3.3Vネイティブ → monkeypadブレイクアウト基板はレギュレータ搭載で5V/3.3V両対応
- SPI通信: SCLK, MOSI, MISO, CS + MOTION割り込みピン

### GPIO拡張 (必要時)
- MCP23017 (I2C) で+16GPIO
- タッチI2Cバスに相乗り → 追加ピン不要
