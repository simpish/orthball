## ピン割り当て (RP2040-Zero)

```
RP2040-Zero (表20ピン + 裏9パッド = 29GPIO)

【表面ピン (20本)】
├── キーマトリクス COL:  GP0, GP1, GP2, GP3, GP4, GP5  (6本)
├── キーマトリクス ROW:  GP6, GP7, GP8, GP9            (4本)
├── PMW3360 SPI0:       GP18(SCK), GP19(MOSI), GP16(MISO), GP17(CS)  (4本)
├── PMW3360 MOTION:     GP20                            (1本)
├── OLED I2C1:          GP14(SDA), GP15(SCL)            (2本)
├── ロータリーエンコーダ: GP26(A), GP27(B), GP28(SW)     (3本)
│
【裏面パッド (9本)】
└── 未使用:             GP10-13, GP21-25                (9本: LED/将来拡張)

使用: 20/29  余り: 9
```

> ※ ピン番号は仮。PCB設計時にI2Cバスの物理ピン制約に合わせて調整する
> ※ OLED変更により GPIO 9本余り → 将来のRGB LED、追加機能に余裕あり

## 技術メモ

### OLED (SSD1306 128×32)
- QMK標準OLEDドライバで対応（カスタムドライバ不要）
- I2C接続（SDA + SCL の2ピンのみ）
- 基板サイズ: 約38×12mm — 親指クラスター中央に配置
- DPI値、レイヤー名、スクロールモード等を表示

### PMW3360接続
- RP2040は3.3Vネイティブ → ブレイクアウト基板はレギュレータ搭載で5V/3.3V両対応
- SPI通信: SCLK, MOSI, MISO, CS + MOTION割り込みピン
- ブレイクアウト基板(28×32mm)はセンサーカップ底に配置、ワイヤーでメインPCBに接続

### GPIO拡張 (必要時)
- MCP23017 (I2C) で+16GPIO
- OLED I2Cバスに相乗り → 追加ピン不要
- OLED変更により9ピン余裕があるため、V1では不要の見込み
